## Definición

La aplicación consiste en una <HighlightedCode>listado de tareas</HighlightedCode> donde se pueden añadir, eliminar y marcar como completadas dichas tareas. Todo utilizando el localStorage para guardar la información en el navegador.

El proyecto se puede dividir en tres partes bien diferenciadas:

- <HighlightedCode>Maquetación</HighlightedCode>

- <HighlightedCode>Estructura/Componentes React</HighlightedCode>

- <HighlightedCode>Lógica/Funcionalidad</HighlightedCode>

### Maquetación

Este proyecto se ha realizado utilizando CSS vanila, permitiendo una mayor libertad a la hora de la personalización y la creación del diseño sin tener que estar límitado a las características de los frameworks actuales.

La maquetación/diseño se ha realizado teniendo en cuenta los siguientes principios:

- <HighlightedCode>Responsive:</HighlightedCode> Uso de <strong>
    media queries
  </strong> para adaptar el diseño a diferentes tamaños de pantallas, ya sean dispostivos
  móviles, tablets o ordenadores de sobremesa.
- <HighlightedCode>Flex-Box:</HighlightedCode> Uso del modelo de cajas <strong>
    flexbox
  </strong> para la distribución de los elementos. Este modelo permite un control
  de la posición y tamaño de los elementos en función de su contenido y su relación
  con otros elementos.{" "}
- <HighlightedCode>Pseudo-clase :root:</HighlightedCode> Está pseudo-clase permite
  definir varibles con valores de propiedades css que se van a estar disponibles
  de forma global en la aplicación.

> El modelo flexbox maneja el <HighlightedCode>layout</HighlightedCode> de elementos en una única dimensión a la vez, ya sea como fila o como columna.

La estructura del proyecto se ha desarrollado utilizando <HighlightedCode>React</HighlightedCode> y se han usado componentes para dividir el proyecto en diferentes partes. Esta técnica o patrón permite una mayor orgnización y escabilidad del proyecto.

### Lógica/Funcionalidad

La lógica de la aplicación se ha desarrollado utilizadno <HighlightedCode>React</HighlightedCode> y <HighlightedCode>ContextAPI</HighlightedCode>.

> <HighlightedCode>ContextAPi</HighlightedCode>
> es una <HighlightedCode>API</HighlightedCode> de <HighlightedCode>
>   React
> </HighlightedCode> que permite el manejo del estado de nuestra aplicación. Esto
> permite que los datos se puedan compartir entre componentes sin tener que pasarlos
> manualmente a través de los <HighlightedCode>props.</HighlightedCode>

Se ha creado un contexto para controlar el estado de la aplicación de forma global, <HighlightedCode>LTProvider.tsx</HighlightedCode>.
Este <strong>Provider</strong> contiene el estado de la aplicación y las funciones para modificarlo. También es necesario crear un contexto <HighlightedCode>LTPContext.tsx</HighlightedCode>, para definir el tipo de datos que va a usar nuestro Provider y el contexto global de la aplicación.

La aplicación cuenta con varias funciones, entre ellas:

- <strong>Obtener el listado de tareas del LocalStorage</strong>
  <br />
  Esta funcionalidad se ejecuta al entrar o recarga la aplicación para obtener
  el listado de tareas guardado en él. Si no existen tareas en localStorage no
  se devuelve ni se obtiene ndad. Para realizar esta Funcionalidad, se utiliza
  el hooks useEffect y se guarda el listado de tareas en un estado llamado tasks
- <strong>Agregar una nueva tarea</strong> <br /> Esta funcionalidad permite añadir
  una nueva tarea al array de tareas. Se hace uso del hooks "useState" para guardar
  el valor del input en un estado llamado "tasks", y una vez finalizada esta funcionalidad,
  se agrega esta nueva tarea al LocalStorage. Esta función también controla si el
  input está vacío o no, y en caso de estar vacío, muestra un mensaje de error a
  través del estado "errorCreateTask".{" "}

* <strong>Eliminar una tarea</strong>
  <br />
  Esta funcionalida permite eliminar una tarea del array de tareas mediante el
  uso de "filter", que crea un nuevo array de tareas sin la tarea seleccionada.
  Una vez finalizada esta funcionaliad, se actualize el LocalStorage con el
  nuevo array de tareas y se actualiza el estado "tasks" con este nuevo array.
  En resumen, la función uso el método "filter" para eliminar la tarea
  seleccionada del estado, se guarda el nuevo array de tareas en el LocalStorage
  y se actualiza el estado "tasks" para reflejar el cambio en la app.
* <strong>Abrir el modal de editar una tarea </strong> <br />
  Esta funcionaliad permite abrir el modal de editar una tarea. Para ello, se
  utiliza el estado "showModal" y se cambia su valor a true. Esto permite que el
  modal se muestre en la aplicación. Esta funcionaliad modifica el estado de la
  tarea seleccionada "stateTask" para que el modal muestre la descripción y el
  identificador de la tarea seleccionada por el usuario.
* <strong>Guardar tarea editada</strong> <br />
  Esta funcionalidad permite guardar la tarea actualizada en el array de tareas.
  Para ello, se hace uso de la descripción agregada en el textarea del modal y
  se la asigna dicho valor a la descripción de la tarea. Todo esto genera un
  nuevo array de tareas que va a ser el que se va a guardar en el LocalStorage y
  se va a utilizar para actualizar el estado "tasks". Por último, se cambia el
  valor del estado "showModal" a false para cerrar el modal.
